Abstract

VG-GUI-Bench evaluates how well multimodal large language models (VLMs) can predict the next GUI action on a mobile device, given reference frames extracted from YouTube tutorial videos and the current screen state. The benchmark is built on the MONDAY dataset. Given a tutorial video showing how to complete a task on a mobile phone, the model must understand the workflow from reference frames, identify its current position in the task from previous action history, and predict the exact next action on the current screen.

Task Formulation

Given a tutorial video, the model predicts the next GUI action by reasoning over:

  1. Understanding the workflow from reference frames (keyframes / uniform samples / annotated frames).
  2. Identifying the current position in the task from previous action history.
  3. Predicting the exact next action on the current screen.

Supported Actions

ActionFormatDescription
CLICKCLICK(x, y)Tap at normalized coordinates (0.0–1.0)
SCROLLSCROLL(x1, y1, x2, y2)Swipe gesture
TYPETYPE("text")Text input
PRESSPRESS("key")BACK / HOME / ENTER
ZOOMZOOM()Multi-touch gesture
FINISHFINISH()Task complete or impossible

Reference Image Modes

The benchmark supports 12 reference modes to study how different visual context strategies affect agent performance.

ModeDescription
singleNo reference frames; only the current screen
originScene-timestamp keyframes from video
gtGround-truth annotated frame with red bounding box (oracle)
annotationAnnotated keyframes as reference
uniform5 / uniform105 or 10 uniformly sampled frames
tasker / bfs / gbfs / dijkstraAlgorithmic keyframe selection strategies
videoagent / videotreeVideo understanding agent-based frame selection

Each mode runs in cut (cropped to phone screen) or nocut (full frame) variant.
